# Xolmis Desktop

[**Xolmis Desktop**](https://github.com/cbeier-studio/Xolmis) is the central hub for managing, validating, analyzing, and archiving the data collected with **Xolmis Mobile**. While the Mobile app is optimized for **fast, offline field data entry**, the Desktop application provides the full suite of tools needed for **long‑term storage, curation, and scientific analysis**.

!!! note
    This manual focuses on **Xolmis Mobile**. For detailed instructions on using **Xolmis Desktop**, refer to the dedicated Desktop documentation.

## Key capabilities

- **Comprehensive data management** – Store, edit, validate, and curate all ornithological records.  
- **Advanced analysis tools** – Generate statistics, reports, summaries, and visualizations.  
- **Structured modules** – Work with surveys, sightings, captures, nests, eggs, specimens, vegetation, weather logs, and more.  
- **Data integrity** – Enforce standardized methods, controlled vocabularies, and hierarchical gazetteers.  
- **Long‑term archiving** – Maintain a stable, centralized database for research projects.  

## Workflow with Xolmis Mobile

1. **Collect data** in the field using Xolmis Mobile (inventories, nests, specimens, notes).  
2. **Export** Mobile data (CSV, Excel, JSON, KML).  
3. **Import into Desktop** using the Mobile Import Assistant.  
4. **Verify and curate** the imported records.  
5. **Analyze, report, and archive** the data in the Desktop environment.  

This workflow ensures that field observations become **validated, standardized, and permanently stored** within a robust research database.

## Best practices

- **Import soon after fieldwork**: Reduces the risk of data loss and keeps the Desktop database up to date.  
- **Review all imported records**: Validate species IDs, locations, counts, and metadata.  
- **Use controlled vocabularies**: Gazetteer, methods, researchers, and institutions should be configured before importing.  
- **Back up regularly**: Desktop databases contain long‑term research data and should be backed up frequently.  

## Relation to Xolmis Mobile

- **Surveys** – Mobile inventories become survey events in Desktop.  
- **Sightings** – Species detections recorded in the field are integrated into the sightings module.  
- **Captures** – Field capture data can be expanded with morphometrics and banding history.  
- **Nests and Eggs** – Monitoring data from Mobile is incorporated into the full reproductive biology workflow.  
- **Specimens** – Field‑collected samples can be linked to laboratory preparations and institutional collections.  

Xolmis Desktop acts as the **analytical and archival counterpart** to the Mobile app, ensuring that field data evolves into structured, high‑quality scientific information.
